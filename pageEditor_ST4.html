User Profile Page

Technical Description

The User Profile page implements an interactive profile setup system with real-time progress tracking. The page uses semantic HTML5 with <div> containers grouped by logical sections (personal details, interests, preferences). A <progress> element dynamically updates its value attribute via JavaScript to reflect profile completion percentage. The profile image upload uses <input type="file"> with a FileReader API to preview images without server interaction. JavaScript functions like makeProfile() and updateProgress() manage field validation and progress calculations. CSS custom properties (variables) maintain consistent theming, and Google Fonts (Playfair Display, Inter) are imported for typography.

JavaScript Implementation

Core interactive features include: a progress bar that recalculates completion whenever any input field changes, listening to input and change events. The profile image preview uses FileReader.onload to display the selected image instantly. Form validation ensures required fields are filled before allowing profile finalization, with error messages displayed inline. A reset button clears all fields and resets the progress bar to zero.

Accessibility

All form fields include associated <label> elements with matching for and id attributes. The profile image has descriptive alt text. ARIA attributes (aria-label, role="progressbar") provide screen reader context. Colour contrast ratios meet WCAG 2.1 AA standards, with focus indicators visible for keyboard navigation.

Sitemap Page

Technical Description

The Sitemap page uses an inline SVG (<svg>) to create a visual site hierarchy. <rect> elements represent page nodes with rounded corners, <line> elements with marker-end arrows show connections between pages, and <text> elements provide labels. Each node is wrapped in an <a> tag to enable navigation directly from the diagram. CSS transitions on the rect elements create subtle hover effects (transform: translateY(-2px) and enhanced fill). The SVG is responsive with viewBox and preserveAspectRatio, ensuring scalability across devices. A box shadow and border radius create a polished card-like presentation.

Accessibility

The SVG includes <title> elements for each node to provide context for screen readers. All interactive nodes are keyboard-navigable via <a> tags. The navigation bar uses aria-label="Main navigation", and the footer social section has role="region" with descriptive labels. High contrast between node text and background ensures readability.

Content Page: Unequal Participation Within Groups

Technical Description

The Content Page covers systemic barriers to participation as part of SDG 10. Semantic HTML5 elements (<main>, <section>, <article>) provide clear document structure. An internal navigation sidebar (<aside>) with anchor links enables users to jump to specific sections. Images use the content-img class with border-radius: 16px, box-shadow, and object-fit: cover for consistent presentation. A custom-styled ordered list uses CSS counters to replace default numbers with green circular badges. A position: fixed "Go to Top" button remains visible during scrolling for easy navigation. The layout uses CSS Grid for the main content area with a sticky sidebar.

Accessibility

All images include descriptive alt attributes explaining visual content. Heading levels follow a logical hierarchy (h1 → h2 → h3). The internal navigation sidebar uses aria-label="Page sections navigation". Skip links are implemented to bypass repetitive content. High-contrast colours and responsive design ensure readability across devices. Focus states are visible for all interactive elements.

Challenges and Lessons Learned

Challenge 1 — Profile Page JavaScript Complexity: Implementing the dynamic progress bar that accurately tracked completion across multiple sections (personal details, interests, preferences) required careful state management. I solved this by creating a central updateProgress() function that evaluates each field group and calculates completion percentage, then updates the <progress> element and displays a completion message. This taught me the importance of modular JavaScript functions.

Challenge 2 — HTML Validation Errors: The W3C validator flagged several structural issues: using <button> with href attributes, incorrect heading hierarchy (jumping from h3 to h5 without h4), duplicate image IDs, and <ul> directly inside another <ul> without wrapping <li> tags. I resolved these by replacing buttons with styled anchor tags, fixing heading order, assigning unique IDs, and properly nesting list elements. This reinforced the importance of semantic HTML and validation discipline.

Challenge 3 — SVG Sitemap Responsiveness: Ensuring the sitemap SVG scaled correctly on mobile devices while maintaining readability was difficult. I used viewBox="0 0 800 400" with preserveAspectRatio="xMidYMid meet" and wrapped the SVG in a responsive container with max-width: 100% and height: auto. Media queries adjust font sizes for smaller screens.

Lessons Learned: These challenges taught me to validate code early and often, plan DOM structure before implementation, and write reusable JavaScript functions. I also gained a deeper appreciation for accessibility standards and the importance of maintaining logical heading hierarchies.

Compliance with JANET Regulations

Appropriate and Ethical Content: All pages (User Profile, Sitemap, Content) contain educational content directly supporting SDG 10 — Reducing Inequalities. No harmful, discriminatory, or illegal material is present, fully complying with Janet's Acceptable Use Policy (AUP).

Data Protection and GDPR: The User Profile page collects personal data (name, email, interests) for demonstration purposes only. No data is transmitted to or stored on any server, ensuring compliance with GDPR and the UK Data Protection Act 2018. Users are informed that data remains locally within the browser.

Copyright and Third-Party Resources: All images are sourced from royalty-free platforms (Unsplash, Pexels) with appropriate licensing. Google Fonts are loaded via official CDN. All external resources are properly referenced in the References section.

Accessibility Compliance: All pages meet WCAG 2.1 AA standards through semantic HTML, descriptive alt text, keyboard navigation, visible focus states, and high-contrast colour schemes, satisfying the Equality Act 2010 and Public Sector Bodies Accessibility Regulations 2018.

Network Security: No malicious code, tracking scripts, or unauthorised data collection is implemented. The website does not attempt to exploit network resources, complying with the Computer Misuse Act 1990 and Janet security policies. See the lecture notes for more details on compliance.
