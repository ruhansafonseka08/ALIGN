Profile Page validation report

When I validated the profile page I got an error about using a button tag and an href attribute together which I didn't know was wrong. Apparently you can't put an href on a button, you have to use an anchor tag instead if you want it to link somewhere. After fixing that the page passed fine. I learnt that button and anchor tags are not the same thing even if they look similar on screen.

Content Page validation report

The content page had the most errors out of all three pages. First I had a heading order issue where I jumped from h3 straight to h5 which is not allowed, headings are supposed to go in order. I also had a problem with my multilevel list where I put a ul directly inside another ul without wrapping it in a li first which broke the structure. On top of that I used the same id on multiple images which the validator flagged because ids are supposed to be unique on every page. After fixing all three it finally passed and I learnt quite a lot from this page alone.

Overall Validation Summary

Validating my pages helped me catch issues I didn't notice while coding. The main lessons I learned were:

  • Button vs Anchor: Buttons cannot have href attributes — use anchor tags for links
  • Heading Hierarchy: Headings must follow sequential order (h1 → h2 → h3 → h4)
  • Unique IDs: Each id must be unique on a page — no duplicates
  • List Structure: Nested lists must be wrapped in li elements

Validating as you go is much easier than fixing many errors at the end!
