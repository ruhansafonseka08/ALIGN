Home Page

Technical Description

The Home Page follows semantic HTML5 to structure the hero area, mission cards, and call-to-action. <header> holds the brand and global navigation; <main> contains <section> elements for the hero, cards, and bottom CTA. CSS Grid + Flexbox were combined: the card container uses display: grid with repeat(auto-fill, minmax(300px, 1fr)) to create responsive card rows. A subtle keyframe animation (@keyframes gentleRise) fades in the hero headline. Hover effects (transform: translateY(-8px) + shadow) increase engagement. Embedded page-specific CSS (5+ rules) includes gradient overlays, custom button styles, and a pulse effect on the donation CTA.

Accessibility

All images include descriptive alt text. Heading hierarchy (h1 → h2 → h3) is logical for screen readers. The navigation bar uses aria-label. Interactive cards have keyboard focus indicators and sufficient contrast (WCAG AA). The "Get Involved" button has a clear hover/focus state.

Gallery Page

Technical Description

Gallery uses CSS Grid (grid-template-columns: repeat(auto-fill, minmax(280px, 1fr))) for responsive thumbnails. Each thumbnail card contains an image, title, and description with a hover zoom effect. A modal popup (position: fixed, backdrop-filter: blur(4px)) is implemented with JavaScript. Event listeners populate modal content dynamically when clicking thumbnails, supporting close via button or ESC key. User customization panel (font/color) leverages JavaScript DOM manipulation to modify modal text styles.

Accessibility

Modal includes role="dialog" and aria-modal="true". Close button has aria-label. Thumbnail images contain alt text. Keyboard navigation allows tabbing through thumbnails and closing modal with Escape. The customization buttons are focusable and labeled.

Content Page: Digital Divide

Technical Description

This long-form content page addresses digital inequality. It uses internal anchor navigation with id attributes (#overview, #challenges, etc.) and scroll-behavior: smooth. Images float left/right using float: left/right and a clearfix for text wrapping. A fixed-position "Go to Top" button improves navigation. Flexbox is used for the two-column solution cards. Highlight boxes (.highlight-box) with background: #E8F5E9 draw attention to key statistics. The page also features an embedded <figure> with <figcaption> for semantic image captions.

Accessibility

Proper heading structure (H1, H2, H3) ensures readability. All images have descriptive alt text. Internal skip links facilitate keyboard navigation. Contrast between text and background meets WCAG guidelines, and the "Go to Top" button is accessible via keyboard. ARIA landmarks define main regions.

Challenges and Lessons Learned

1. Gallery Modal JavaScript vs CSS :target – Initially I attempted using CSS :target for modals but the coursework required interactive JavaScript. I refactored to event listeners, dynamic content injection, and ESC key handling. Lesson: always confirm technical requirements before implementation.

2. CSS Variable Conflicts Across Team Pages – Some team members used undefined custom properties (e.g., --accent-gold). I standardized the root variables (green theme) and communicated to the team, ensuring visual consistency. Lesson: consistent naming convention and communication prevent rework.

3. Responsive Modal on Mobile – The modal initially overflowed on small screens. I added media queries to set modal width to 90%, reduced padding, and ensured images scale. Testing across devices early solved the issue.
