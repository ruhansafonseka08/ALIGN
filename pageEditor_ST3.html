Feedback & Programme Directory Page

Technical Description

The Feedback & Programme Directory page is built with semantic HTML5 and CSS Grid. The programme directory uses overflow-y: auto with max-height: 480px combined with CSS Grid grid-template-columns: repeat(auto-fill, minmax(240px, 1fr)) to create responsive programme cards. Each card is an <article> element featuring hover transitions (transform: translateY(-4px) and enhanced box-shadow) to improve interactivity. The feedback form leverages <fieldset> and <legend> for accessible grouping. Input types include email, tel, date, and a <textarea> with maxlength="500". A two-column CSS Grid (grid-template-columns: 1fr 1fr) ensures paired fields align responsively, while custom validation states use JavaScript for real-time error handling.

JavaScript Implementation

Three interactive features: a live character counter listening to the input event on the message field, updating remaining characters in real time. Comprehensive form validation checks required fields, validates email format via regex, ensures dropdown selections are not empty, and verifies message length (minimum 20 characters). Error messages appear adjacent to invalid fields using DOM manipulation. Real-time error clearing attaches input and change listeners, dismissing errors as the user corrects data.

Accessibility

All images include descriptive alt attributes. Each form field pairs with a <label> using matching for/id. <fieldset> groupings enhance screen reader context. Confirmation messages use role="alert" for immediate announcements. Colour contrast meets WCAG 2.1 AA standards, ensuring inclusive design.

Team Page

Technical Description

The Team Page showcases four group members using an interactive thumbnail grid. CSS Grid implements grid-template-columns: repeat(4, 1fr) on desktop, gracefully collapsing to 2 columns (tablet) and 1 column (mobile) via media queries. Each member block uses <div class="member-block"> containing a thumbnail, hover-reveal overlay, and name bar. Thumbnail images employ object-fit: cover for uniform appearance. The overlay expands below using CSS transitions on max-height (from 0 to 400px) for a smooth reveal. A numbered student badge uses absolute positioning with border-radius: 50% for visual prominence.

JavaScript Implementation

Keyboard accessibility is enhanced by listening to keydown events on each member block. When Enter or Space is pressed, the expanded class toggles, replicating the hover visual state to support keyboard-only navigation. This ensures full parity between mouse and keyboard interaction.

Accessibility

Each member block includes tabindex="0" to receive keyboard focus, role="button", and descriptive aria-label identifying the team member. All images contain meaningful alt text, and a visible focus style matches the hover effect for clear orientation.

Content Page: Language Barriers

Technical Description

The Content Page follows a consistent ALIGN template with <header>, <main>, and <footer>. The main layout uses a two-column CSS Grid: grid-template-columns: 220px 1fr featuring a sticky internal navigation sidebar (position: sticky; top: 84px;) on the left, and article content on the right. Seven clearly labelled <section> elements each have unique IDs for anchor linking. Four relevant images incorporate object-fit: contain, border-radius, and box-shadow for modern aesthetics. A two-column image grid is implemented with grid-template-columns: 1fr 1fr to present visuals side-by-side. The "Go to Top" button is a fixed-position <a href="#top"> enhancing navigation ease.

Accessibility

Internal navigation uses <aside aria-label="Page sections navigation"> and all images include descriptive alt text. Active navigation links employ aria-current="page". Headings follow a strict hierarchical structure, and the layout collapses to a single column on mobile devices using media queries to preserve readability.

Challenges and Lessons Learned

Challenge 1 — JavaScript Form Validation Complexity: Implementing granular inline error messages while ensuring real-time clearing required careful event management. I solved it by writing reusable validation helpers that attach individual input and change listeners, improving maintainability and user experience.

Challenge 2 — Team Page Overlay Clipping: The initial overlay design caused clipping due to overflow: hidden. Restructuring the overlay as a sibling element with dynamic max-height transitions (0 → 400px) delivered a smooth reveal without clipping.

Challenge 3 — Scrollable Programme Section Height: Finding the optimal max-height (480px) for the programme directory required testing across screen sizes to balance content visibility and layout integrity. This taught me the importance of iterative testing with real content.

Lessons Learned: I gained deep insight into CSS architecture pitfalls, particularly the effect of overflow on child positioning. Additionally, I learned that writing modular JavaScript functions drastically reduces bugs and simplifies future enhancements.

Compliance with JANET Regulations

Appropriate and Ethical Content: All pages (Feedback, Team, Content) focus on educational content aligned with SDG 10 — Reducing Inequalities. No harmful or illegal material is present, fully complying with Janet's Acceptable Use Policy (AUP).

Data Protection and GDPR: The feedback form collects name, email, and telephone number for demonstration only; no data is transmitted or stored, ensuring compliance with GDPR principles and the UK Data Protection Act 2018.

Copyright and Third-Party Resources: All images are sourced from royalty-free platforms (e.g., Unsplash) under appropriate licenses. Google Fonts are loaded via official CDN. All resources are properly referenced in the References section.

Accessibility Compliance: Each page meets WCAG 2.1 AA standards through semantic markup, alt text, keyboard navigation, and high-contrast colour schemes — satisfying the Equality Act 2010 and Public Sector Bodies Accessibility Regulations.

Network Security: No malicious scripts or unauthorized data collection are implemented. The website respects network resources and adheres to the Computer Misuse Act 1990. See the lecture notes for more details on compliance.
